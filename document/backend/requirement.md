# 促销活动管理标准化系统 - 需求文档

## 1. 需求概述

### 1.1 业务背景
本系统为促销活动全生命周期管理平台，核心特征为**状态驱动业务、事件触发流转**。系统包含两套独立且联动的状态机：
- **活动状态机**：控制活动生命周期（草稿→待生效→生效中→过时/下线→结束）
- **审核状态机**：控制审批流程（等待审核→审核中→通过/驳回/不通过/作废→结束）

### 1.2 架构设计原则
- **事件驱动架构（EDA）**：所有业务行为均产生标准化事件，系统通过监听事件触发状态流转
- **DDD领域驱动设计**：模块划分遵循领域边界，确保高内聚低耦合
- **双状态机联动**：审核状态机为前置驱动，活动状态机为生命周期载体

## 2. 功能需求

### 2.1 活动管理模块

| 功能点 | 需求描述 | 对应事件 | 状态流转 |
| :--- | :--- | :--- | :--- |
| 创建活动 | 管理员创建促销活动草稿 | `E_CREATE_DRAFT` | 活动: 开始→草稿; 审核: 开始→等待审核 |
| 提交审核 | 管理员提交活动进行审核 | `E_SUBMIT_AUDIT` | 审核: 等待审核→审核中 |
| 更新活动 | 管理员修改活动信息（仅草稿状态） | `E_UPDATE_ACTIVITY` | 保持当前状态 |
| 删除活动 | 管理员删除活动（先下线再删除） | `E_DELETE_ACTIVITY` | 活动: 任意→下线→结束 |
| 手动下线 | 管理员强制下线生效中活动 | `E_MANUAL_OFFLINE` | 活动: 生效中→下线→结束 |

### 2.2 sku管理模块

| 功能点   | 需求描述              | 对应事件           | 状态流转 |
|:------|:------------------|:---------------|:-----|
| 创建sku | 管理员创建符合条件的sku     | `E_CREATE_SKU` | -    |
| 删除sku | 管理员删除指定sku        | `E_DELETE_SKU` | -    |
| 更新sku | 管理员更新指定id的sku     | `E_UPDATE_SKU` | -    |
| 查看sku | 查看指定id列表中的所有sku信息 | `E_LIST_SKU`   | -    |

### 2.3 审核流程模块

| 功能点 | 需求描述 | 对应事件 | 状态流转 |
| :--- | :--- | :--- | :--- |
| 审核通过 | 审核员批准活动 | `E_AUDIT_PASS` | 审核: 审核中→通过; 活动: 草稿→待生效 |
| 审核驳回 | 审核员驳回活动（可修改重提） | `E_AUDIT_REJECT` | 审核: 审核中→驳回; 活动: 保持草稿 |
| 审核不通过 | 审核员拒绝活动（终止） | `E_AUDIT_NOTPASS` | 审核: 审核中→不通过; 活动: 草稿→结束 |
| 审核作废 | 审核员作废活动 | `E_AUDIT_CANCEL` | 审核: 等待审核/驳回→作废; 活动: 草稿→结束 |

### 2.4 定时任务模块

| 功能点 | 需求描述 | 对应事件 | 状态流转 |
| :--- | :--- | :--- | :--- |
| 自动生效 | 到达活动开始时间自动生效 | `E_ACTIVE_ONLINE` | 活动: 待生效→生效中 |
| 自动过期 | 到达活动结束时间自动过期 | `E_ACTIVE_EXPIRE` | 活动: 生效中→过时→结束 |

### 2.5 查询模块

| 功能点 | 需求描述 |
| :--- | :--- |
| 活动列表查询 | 支持按状态、时间、名称等条件查询 |
| 活动详情查询 | 查询单个活动完整信息 |
| 审核状态查询 | 查询活动审核进度 |
| 事件日志查询 | 查询活动所有事件记录 |

## 3. 非功能需求

| 类别 | 需求描述 |
| :--- | :--- |
| **可靠性** | 事件可追溯、可重试、可回放 |
| **一致性** | 双状态机状态保持一致，禁止非法流转 |
| **扩展性** | 新增促销规则、审核节点仅需新增事件监听 |
| **可维护性** | 状态变更统一由事件驱动，杜绝硬编码直接修改 |

## 4. 核心事件模型

### 4.1 事件枚举定义

| 事件编码 | 事件名称 | 触发角色 | 核心作用 |
| :--- | :--- | :--- | :--- |
| `E_CREATE_DRAFT` | 创建草稿事件 | 管理员 | 生成草稿活动，初始化双状态机 |
| `E_SUBMIT_AUDIT` | 提交审核事件 | 管理员 | 审核状态由等待审核→审核中 |
| `E_AUDIT_PASS` | 审核通过事件 | 审核员 | 审核完结，活动由草稿→待生效 |
| `E_AUDIT_REJECT` | 审核驳回事件 | 审核员 | 审核驳回，活动保持草稿 |
| `E_AUDIT_NOTPASS` | 审核不通过事件 | 审核员 | 审核终止，活动终结 |
| `E_AUDIT_CANCEL` | 审核作废事件 | 审核员 | 审核作废，活动终结 |
| `E_ACTIVE_ONLINE` | 活动自动生效事件 | 定时任务 | 活动由待生效→生效中 |
| `E_ACTIVE_EXPIRE` | 活动过期事件 | 定时任务 | 活动由生效中→过时→结束 |
| `E_MANUAL_OFFLINE` | 手动下线事件 | 管理员 | 生效中活动强制下线 |
| `E_UPDATE_ACTIVITY` | 活动更新事件 | 管理员 | 更新活动信息，触发规则校验 |
| `E_DELETE_ACTIVITY` | 活动删除事件 | 管理员 | 先下线再删除 |

### 4.2 事件结构定义

```json
{
  "eventId": "string (UUID)",
  "eventType": "string (事件编码)",
  "promotionId": "string (活动ID)",
  "prevActivityStatus": "int (前置活动状态)",
  "prevAuditStatus": "int (前置审核状态)",
  "operator": "string (操作人ID)",
  "eventTime": "datetime (事件时间)",
  "params": "object (事件参数)"
}
```

## 7. 状态码定义

## 8. DDD模块划分

### 8.1 领域边界

| 领域模块 | 职责说明 | 核心实体 |
| :--- | :--- | :--- |
| **促销活动域** | 活动生命周期管理 | Promotion, PromotionSku |
| **审核流程域** | 审批流程管理 | AuditRecord |
| **事件总线域** | 事件生产、分发、路由 | Event, EventLog |
| **用户域** | 用户认证与权限 | User, Role |

### 8.2 模块交互关系

```
用户域 ──触发──→ 促销活动域 ──产生事件──→ 事件总线域 ──路由──→ 审核流程域
                                    ──路由──→ 促销活动域（状态更新）
```

## 9. 状态机流转约束

1. **事件驱动原则**：所有状态变更必须由事件触发，禁止直接UPDATE状态字段
2. **合法性校验**：状态机引擎前置校验流转合法性，非法事件直接拦截
3. **终态保护**：已进入终态的双状态机，禁止再次触发任何变更事件
4. **审核前置**：审核未完成的活动，无法触发生效、过期、下线等生命周期事件
